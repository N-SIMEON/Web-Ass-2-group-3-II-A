# AI Usage Declaration — AgriStack

## Overview
This document describes how AI tools (specifically Claude by Anthropic) were used during the development of AgriStack. In line with academic integrity standards, AI was used as a **helper and accelerator**, not as a replacement for understanding and original work.

---

## How AI Was Used

### ✅ Acceptable Uses

| Area | How AI Helped | Team Responsibility |
|------|--------------|---------------------|
| Boilerplate code structure | Suggested MVC folder structure pattern | Team reviewed, adapted, and extended |
| CSS patterns | Helped debug flexbox/grid layout issues | Team designed the visual system and color palette |
| SQL schema review | Checked for normalization issues | Team designed all tables and relationships |
| Error debugging | Helped trace PHP session errors | Team understood root cause and fixed |
| Documentation templates | Suggested markdown table structure | All content (problem, user stories, etc.) was written by team based on real field research |
| README formatting | Suggested setup step structure | Team wrote all actual setup instructions |

### ❌ Not Used For
- Writing business logic (controllers, model queries)
- Generating user stories (based on actual farmer interviews)
- Making design decisions (color palette, layout, typography)
- Security implementations (password hashing, prepared statements)
- Problem statement (from real field observation in Musanze)

---

## Field Research Note
The problem statement, stakeholder analysis, and user stories are based on:
- Direct interviews with 3 potato farmers in Kinigi Sector, Musanze
- Market observation at Musanze Central Market (March 2024)
- Consultation with Musanze District Agricultural Officer
- Research on Rwanda potato supply chain (RAB 2023 report)

**AI did not conduct or fabricate this research.** All insights are from the team's direct engagement with the problem domain.

---

## Conclusion
AI tools were used responsibly to accelerate development while ensuring every team member understood what was built and why. The conceptual work, research, design decisions, and core implementations are the original work of the AgriStack development team.
